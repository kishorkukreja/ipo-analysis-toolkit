# IPO Domain Conventions

## Market Scope

Use these skills for Indian IPOs on NSE/BSE mainboard, NSE Emerge, and BSE SME. Do not apply US IPO assumptions unless the user explicitly asks for a foreign IPO.

## Timeline Defaults

- Current default listing timeline: T+3.
- Do not describe T+6 as the current default.
- If a source or old plan says T+6, mark it as historical or stale and apply the current T+3 framework unless the issue date proves otherwise.

## SME vs Mainboard

Always identify whether the IPO is mainboard or SME. If unknown, treat classification as a required lookup.

Mainboard analysis should emphasize institutional demand, peer valuation, category-wise subscription, and retail allotment odds.

SME analysis must add liquidity risk, market maker context, higher lot-value risk, exchange platform distinction, and the July 2025 process changes for issues opening on or after 2025-07-01.

## Financial Formatting

- Use INR and Indian number formatting when values are known.
- Use `Rs` in plain ASCII text unless copying a source title.
- State whether numbers are in crore, lakh, or per-share terms.
- Indian fiscal years run April to March.

## Safety Boundary

Skills can provide structured decision support, scenarios, and risk framing. They must not guarantee allotment, listing gain, GMP reliability, or investment returns. Final outputs should be framed as research support, not personal financial advice.
