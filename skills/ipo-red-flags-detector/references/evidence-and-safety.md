# Evidence and Safety Reference

## Source Hierarchy

Use sources in this order:

1. Current DRHP/RHP/prospectus/abridged prospectus, preferably exchange-hosted or SEBI-hosted.
2. SEBI regulations, circulars, board decisions, orders, and investor advisories.
3. NSE/BSE issue pages, SME platform pages, subscription filings, basis of allotment, and market-maker references.
4. Issuer disclosures, annual reports, MCA filings, and registrar documents.
5. Reputable financial press and IPO portals only as corroboration or search aids.

Never let a secondary source override the offer document or official regulatory/exchange source.

## Required Evidence Per Flag

Every flag should include:

- Source URL and document section/page when available.
- Metric and denominator, for example OFS/issue size, OCF/PAT, RPT sales/revenue, litigation/net worth.
- Threshold that triggered severity.
- Severity: Critical, Major, Minor, or Info.
- Confidence: High for direct offer-document/regulator evidence, Medium for official source with missing page context or corroborated secondary evidence, Low for secondary-source-only or inferred evidence.
- One-sentence investor implication.
- What would reduce or disprove the concern.

## Missing Data Rules

If DRHP/RHP/prospectus is unavailable:

- Do not issue a strong clean verdict.
- Use secondary sources only for a provisional scan.
- Set Confidence to Low.
- List missing sections and ask for the DRHP/RHP/prospectus URL or PDF.
- Apply a data gap penalty of +1 to +3 points.

If a section is present but amounts are missing:

- Mark the exposure as unquantified.
- Do not assume it is immaterial.
- State what denominator is needed: net worth, revenue, cash, issue size, or profit.

## Safety Language

Use direct but bounded language:

- "This is a document-backed red flag" when evidence is from the offer document or regulator.
- "This is provisional" when based on secondary sources.
- "Avoid stance from red-flag lens" for auto-reject cases.
- "Not investment advice" when the user asks for apply/avoid or personalized allocation.

Avoid:

- Guaranteed returns, listing gains, allotment outcomes, or loss avoidance.
- Treating GMP as official or decisive.
- Defamatory claims about fraud unless citing an official order and clearly attributing the finding.
- Strong clean conclusions when source coverage is incomplete.

## Current Timeline Handling

Use T+3 as the current default Indian IPO cycle. If a source says T+6, treat it as historical/stale unless the issue date proves the IPO followed that timeline. State dates as YYYY-MM-DD when timeline or SME process treatment matters.
